## 3. Acceptance Criteria (Must Pass)

### 3.1 Feature F-006: Probe Frame Definitions

- [ ] **F006-P01**: Package `pkg/transport/probes` compiles — exit code 0
- [ ] **F006-P02**: `PingFrame` struct exported with fields: `Type uint8`, `Seq uint64`, `SentAt int64` (Unix nanoseconds)
- [ ] **F006-P03**: `PongFrame` struct exported with fields: `Type uint8`, `Seq uint64`, `EchoSentAt int64`, `ReceivedAt int64`
- [ ] **F006-P04**: `EncodePing(seq uint64, sentAt time.Time) []byte` produces a fixed-size wire frame
- [ ] **F006-P05**: `DecodePong(buf []byte) (*PongFrame, error)` parses the wire frame and returns error on truncated input

### 3.2 Feature F-006: ProbeRecord

- [ ] **F006-R01**: `ProbeRecord` struct exported in `pkg/driver/pathmgr` with fields: `RTTSampleEWMA int64` (nanoseconds, atomic), `LossRate uint32` (fixed-point, parts per million), `ThroughputBytesPerSec int64` (atomic)
- [ ] **F006-R02**: EWMA RTT update uses α = 0.125 (per RFC 6298 sRTT formula): `sRTT = (1 - α) * sRTT + α * rttSample`
- [ ] **F006-R03**: All `ProbeRecord` fields updated via `sync/atomic` — no mutex on the update path
- [ ] **F006-R04**: When PONG not received within `probeTimeoutMs` (default 500 ms), `RTTSampleEWMA` is set to `math.MaxInt64` so the Arbitrator excludes this connection from LowestRTT selection

### 3.3 Feature F-006: PathManager Agent

- [ ] **F006-M01**: Package `pkg/driver/pathmgr` compiles — exit code 0
- [ ] **F006-M02**: `PathManager` struct exported with `Run(ctx context.Context) error` method
- [ ] **F006-M03**: PathManager accepts a pool snapshot at construction (or via `SetPool(snapshot PoolSnapshot)`) — does not import hsd process state directly
- [ ] **F006-M04**: PathManager sends a `PingFrame` on each live connection every `probeIntervalMs` (default 100 ms)
- [ ] **F006-M05**: PathManager receives `PongFrame` responses and matches them to outstanding PINGs by `Seq` number
- [ ] **F006-M06**: `PoolSnapshot` type exposed with `Connections() []Connection` returning a point-in-time copy — updated atomically when pool membership changes
- [ ] **F006-M07**: PathManager exposes `ProbeRecords() map[string]*ProbeRecord` (keyed by connection ID) for Arbitrator consumption
- [ ] **F006-M08**: PathManager shuts down cleanly when ctx is cancelled — no goroutine leak (`goleak.VerifyNone`)

### 3.4 Tests

- [ ] **F006-T01**: `go test -race -count=1 ./pkg/driver/pathmgr/... ./pkg/transport/probes/...` exits 0
- [ ] **F006-T02**: Combined coverage ≥ 90%
- [ ] **F006-T03**: Test `TestPathManager_EWMARTTUpdate` verifies: inject 5 PONG frames with known RTT values; verify `RTTSampleEWMA` converges per α=0.125 formula within 0.1% tolerance
- [ ] **F006-T04**: Test `TestPathManager_TimeoutExcludes` verifies: inject a PING with no corresponding PONG for 600 ms; verify `RTTSampleEWMA` is set to `math.MaxInt64`
- [ ] **F006-T05**: Test `TestPathManager_GracefulShutdown` starts PathManager, cancels ctx, verifies all goroutines exit within 500 ms

## 6. Holdout Scenarios

| Scenario ID | Description | Expected Outcome |
|---|---|---|
| S4-H01 | 4-connection pool; one connection stops responding to PINGs | That connection's RTT set to MaxInt64; other connections unaffected |
| S4-H02 | PONG with unknown Seq number received | PONG silently discarded; no panic; unknown seq counter incremented |
| S4-H03 | Pool snapshot updated mid-probe-loop (connection removed) | PathManager detects stale entry; sends no probe to removed connection |
| S4-H04 | ctx cancelled while 8 probes are in-flight | PathManager exits cleanly; no goroutine leak |
