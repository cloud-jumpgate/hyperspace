## 3. Acceptance Criteria (Must Pass)

### 3.1 Feature F-001: Log Buffer

#### Compilation and structure

- [ ] **F001-C01**: Package `pkg/logbuffer` compiles with `go build ./pkg/logbuffer/...` — exit code 0, no errors
- [ ] **F001-C02**: Package exports the following types: `LogBuffer`, `Appender`, `Reader`, `FragmentHandler` (function type `func(buf []byte, header *Header)`)
- [ ] **F001-C03**: `Header` struct has exactly these exported fields: `FrameLength int32`, `Flags uint8`, `FrameType uint8`, `StreamID int32`, `SessionID int32`, `ReservedValue int64`
- [ ] **F001-C04**: Frame header total size is 24 bytes — verified by `unsafe.Sizeof(Header{}) == 24` in a test

#### Log buffer construction and mmap

- [ ] **F001-F01**: `NewLogBuffer(path string, termLength int64) (*LogBuffer, error)` creates a mmap'd file at `path` with permission `0600`
- [ ] **F001-F02**: The mmap'd region size equals `termLength * 3` plus a metadata section of at least 64 bytes (for term tail counters and active term ID)
- [ ] **F001-F03**: `LogBuffer.Close() error` unmaps the region and closes the file without error on a cleanly used buffer

#### Appender

- [ ] **F001-F04**: `LogBuffer.NewAppender() (*Appender, error)` returns an Appender bound to the buffer
- [ ] **F001-F05**: `Appender.Append(buf []byte) (position int64, err error)` writes the frame header + payload atomically to the active term using a single `sync/atomic` CAS on the tail counter — no mutex used in the append path
- [ ] **F001-F06**: `Appender.Append` returns `ErrBackPressure` (sentinel error, not a wrapped error) when the active term is full and the next term is not yet clean
- [ ] **F001-F07**: When the active term fills and the next term is clean, `Appender.Append` atomically rotates to the next term (increments `activeTermID` mod 3) and writes the frame to the new term
- [ ] **F001-F08**: `Appender.Append` is safe to call concurrently from multiple goroutines — verified by `go test -race`

#### Reader (poll loop)

- [ ] **F001-F09**: `LogBuffer.NewReader() (*Reader, error)` returns a Reader at position 0
- [ ] **F001-F10**: `Reader.Poll(handler FragmentHandler, limit int) int` invokes `handler` for each complete frame at the current read position, up to `limit` frames, and advances the reader position
- [ ] **F001-F11**: `Poll` passes a slice directly into the mmap region to `handler` — no intermediate copy allocation (verified by zero allocations in benchmark: `go test -bench=BenchmarkPoll -benchmem` shows 0 allocs/op)
- [ ] **F001-F12**: `Poll` returns 0 when no new frames are available without blocking

#### Tests

- [ ] **F001-T01**: `go test -race -count=1 ./pkg/logbuffer/...` exits 0 — all tests pass, no race conditions
- [ ] **F001-T02**: `go test -coverprofile=cover.out ./pkg/logbuffer/...` produces ≥ 90% statement coverage
- [ ] **F001-T03**: Test `TestAppender_ThreeTermRotation` verifies: fill term 0, verify rotation to term 1, fill term 1, verify rotation to term 2, fill term 2, verify `ErrBackPressure` until term 0 is consumed
- [ ] **F001-T04**: Test `TestAppender_ConcurrentWrites` spawns 8 goroutines each calling `Append` 10,000 times; no race, no data corruption, all frames recoverable by Reader
- [ ] **F001-T05**: Test `TestLogBuffer_FilePermissions` verifies the created file has mode `0600`

### 3.2 Feature F-002: Ring Buffers and IPC

#### Package structure

- [ ] **F002-C01**: Package `pkg/ipc/ringbuffer` compiles — exit code 0
- [ ] **F002-C02**: Package `pkg/ipc/broadcast` compiles — exit code 0
- [ ] **F002-C03**: Package `pkg/ipc/memmap` compiles — exit code 0

#### SPSC Ring Buffer

- [ ] **F002-F01**: `ringbuffer.NewSPSC(size int) (*SPSCRingBuffer, error)` creates a ring buffer backed by a mmap'd region of `size` bytes
- [ ] **F002-F02**: `SPSCRingBuffer.Write(msg []byte) error` returns `ErrFull` when the buffer is full — does not block
- [ ] **F002-F03**: `SPSCRingBuffer.Read() ([]byte, error)` returns `ErrEmpty` when no data is available — does not block
- [ ] **F002-F04**: SPSC ring uses only `sync/atomic` operations for producer and consumer positions — no mutex (verified by code review criterion: `go vet ./pkg/ipc/ringbuffer/...` passes and grep for `sync.Mutex` in `spsc.go` returns no matches)

#### MPSC Ring Buffer

- [ ] **F002-F05**: `ringbuffer.NewMPSC(size int) (*MPSCRingBuffer, error)` creates a multi-producer ring buffer
- [ ] **F002-F06**: `MPSCRingBuffer.Write(msg []byte) error` is safe to call concurrently from multiple goroutines — verified by `go test -race`
- [ ] **F002-F07**: `MPSCRingBuffer.Read() ([]byte, error)` is called from a single consumer goroutine only (documented in godoc); concurrent reads are not supported

#### Broadcast

- [ ] **F002-F08**: `broadcast.NewTransmitter(size int) (*Transmitter, error)` creates a broadcast buffer
- [ ] **F002-F09**: `broadcast.NewReceiver(t *Transmitter) *Receiver` creates a receiver subscribed to the transmitter
- [ ] **F002-F10**: `Transmitter.Transmit(msg []byte) error` makes `msg` available to all current Receivers
- [ ] **F002-F11**: `Receiver.Receive() ([]byte, bool)` returns the next available message and `true`, or `nil, false` if no new message

#### Mmap utilities

- [ ] **F002-F12**: `memmap.Map(path string, size int64, perm os.FileMode) ([]byte, error)` creates or opens a file, truncates to `size`, and returns an mmap'd slice
- [ ] **F002-F13**: `memmap.Unmap(data []byte) error` unmaps the region — no error on a cleanly mapped region
- [ ] **F002-F14**: `memmap.Map` creates files with the specified `perm` (caller is responsible for passing `0600`)

#### Tests

- [ ] **F002-T01**: `go test -race -count=1 ./pkg/ipc/...` exits 0 — all tests pass, no race conditions
- [ ] **F002-T02**: `go test -coverprofile=cover.out ./pkg/ipc/...` produces ≥ 90% statement coverage
- [ ] **F002-T03**: Test `TestMPSC_ConcurrentProducers` spawns 16 goroutines writing 1,000 messages each; single consumer reads all 16,000 messages without error or data corruption
- [ ] **F002-T04**: Test `TestBroadcast_MultipleReceivers` verifies 4 receivers all receive every transmitted message

### 3.3 Non-Functional Criteria (Both Features)

- [ ] **NF-01**: `golangci-lint run ./pkg/logbuffer/... ./pkg/ipc/...` exits 0 — zero lint errors
- [ ] **NF-02**: `go build ./...` exits 0 — entire module compiles cleanly
- [ ] **NF-03**: `go test -race ./...` exits 0 — no races anywhere in the module
- [ ] **NF-04**: No `sync.Mutex` used in `Appender.Append`, `SPSCRingBuffer.Write`, or `SPSCRingBuffer.Read` — these paths must be lock-free
- [ ] **NF-05**: All mmap file paths created in tests use `t.TempDir()` — no test files leak to the filesystem after test completion
- [ ] **NF-06**: Benchmark `BenchmarkAppender` exists and demonstrates > 5 million appends/second on the CI runner (informational — not a hard gate, but must compile and run)

---

### 3.4 Security Criteria

- [ ] **SEC-01**: All mmap files created by `pkg/ipc/memmap.Map` when `perm=0600` are not world-readable — verified by `TestMmap_FilePermissions`
- [ ] **SEC-02**: No hardcoded file paths in `pkg/logbuffer` or `pkg/ipc` — all paths are parameters
- [ ] **SEC-03**: Frame length field in `Header` is validated against a maximum value before the Reader allocates or slices any payload buffer

## 4. Definition of Done

The sprint is marked complete and features F-001 and F-002 are set to `in_review` in `progress.json` ONLY when:

1. All acceptance criteria above are met and verified
2. `go test -race -coverprofile=cover.out ./pkg/logbuffer/... ./pkg/ipc/...` exits 0 with ≥ 90% coverage
3. `golangci-lint run ./...` exits 0
4. `go build ./...` exits 0
5. Code Evaluator returns PASS
6. `progress.json` updated to `in_review` by implementing agent (→ `passing` after Evaluator PASS)

## 5. Holdout Scenarios

| Scenario ID | Description | Expected Outcome |
|---|---|---|
| S1-H01 | Appender writes exactly `termLength - 24` bytes to fill term 0; next Append triggers rotation | Term 1 becomes active; no data loss; previous data readable from term 0 |
| S1-H02 | 32 concurrent goroutines call `Appender.Append` with the race detector enabled | No race condition; all frames written; goroutines complete without deadlock |
| S1-H03 | `Reader.Poll` called on an empty log buffer | Returns 0; handler not invoked; no panic |
| S1-H04 | `SPSCRingBuffer.Write` called until `ErrFull`; then `Read` drains all; then Write again | All writes succeed after drain; no memory corruption |
| S1-H05 | `memmap.Map` called with path to a file in a non-existent directory | Returns a descriptive error; no panic; no partial file created |
| S1-H06 | `MPSCRingBuffer` with 16 concurrent producers; consumer verifies message count equals 16 * N | All messages received; no duplicates; no missing messages |

Minimum satisfaction score to pass: **0.95** (≥ 95% of scenarios must pass)

## 6. Out of Scope

The following are explicitly out of scope for Sprint S1:

- Network transport of any kind — log buffers are local shared memory only
- QUIC or UDP — not in this sprint
- Driver agents (Conductor, Sender, Receiver) — Sprint S3
- Client library API — Sprint S6
- AWS integrations — Sprint S9
- Term recycling policy beyond the three-term rotation defined in F-001
- Cross-host IPC — ring buffers are intra-host only
