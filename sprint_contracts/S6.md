## 3. Acceptance Criteria (Must Pass)

### 3.1 Feature F-010: Channel URI Parser

- [ ] **F010-U01**: Package `pkg/channel` compiles — exit code 0
- [ ] **F010-U02**: `Parse(uri string) (*ChannelParams, error)` accepts URIs of the form `hs:quic?endpoint=host:port|pool=N|cc=algo`
- [ ] **F010-U03**: `ChannelParams` struct has fields: `Scheme string`, `Transport string`, `Endpoint string`, `PoolSize int`, `CC string`
- [ ] **F010-U04**: `Parse` returns a descriptive error for: missing `endpoint` param, non-integer `pool` value, unknown scheme (not `hs`)
- [ ] **F010-U05**: `Parse` defaults: `PoolSize = 4`, `CC = "bbrv3"` when those params are absent
- [ ] **F010-U06**: `pkg/channel` test coverage ≥ 90%

### 3.2 Feature F-010: Client

- [ ] **F010-C01**: Package `pkg/client` compiles — exit code 0
- [ ] **F010-C02**: `NewClient(ctx context.Context, driverPath string) (*Client, error)` opens the CNC mmap file at `driverPath`, verifies the driver heartbeat is live, and returns a connected Client
- [ ] **F010-C03**: `Client.AddPublication(ctx context.Context, channel string, streamId int32) (*Publication, error)` sends an `AddPublication` command to the Conductor ring buffer and blocks until acknowledgement or ctx deadline
- [ ] **F010-C04**: `Client.AddSubscription(ctx context.Context, channel string, streamId int32) (*Subscription, error)` sends an `AddSubscription` command to the Conductor ring buffer and blocks until acknowledgement or ctx deadline
- [ ] **F010-C05**: `Client.Close() error` sends a `ClientClose` command and releases all mmap resources
- [ ] **F010-C06**: `Client` returns `ErrDriverUnavailable` from `NewClient` if the CNC heartbeat timestamp is stale by more than `clientLivenessTimeout` (default 1 s)

### 3.3 Feature F-010: Publication

- [ ] **F010-P01**: `Publication.Offer(buf []byte) (int64, error)` writes `buf` to the log buffer's active term using `Appender.Append`
- [ ] **F010-P02**: `Offer` returns the log buffer position (int64) on success
- [ ] **F010-P03**: `Offer` returns `ErrBackPressure` (not a wrapped error) when the log buffer is full — identical sentinel to the one returned by `Appender.Append`
- [ ] **F010-P04**: `Offer` does not block — it is a non-blocking call that returns immediately with `ErrBackPressure` when the buffer is full
- [ ] **F010-P05**: `Publication.Close() error` sends `RemovePublication` to Conductor and releases the log buffer mmap

### 3.4 Feature F-010: Subscription and Image

- [ ] **F010-S01**: `Subscription.Poll(handler FragmentHandler, limit int) int` polls the image log buffer for up to `limit` frames and invokes `handler` for each
- [ ] **F010-S02**: `FragmentHandler` type defined as `func(buf []byte, header *logbuffer.Header)` — handler receives a slice directly into the mmap region (zero-copy)
- [ ] **F010-S03**: `Subscription.Poll` returns 0 when no new fragments are available without blocking
- [ ] **F010-S04**: `Image` type represents the receive-side log buffer backing a Subscription; not directly constructed by callers
- [ ] **F010-S05**: `broadcast.NewReceiver` (current-tail) is used for all broadcast subscriptions — `NewReceiverFromStart` must not be used (to avoid replaying pre-connection events)

### 3.5 Tests

- [ ] **F010-T01**: `go test -race -count=1 ./pkg/client/... ./pkg/channel/...` exits 0
- [ ] **F010-T02**: Combined coverage ≥ 90%
- [ ] **F010-T03**: Integration test `TestClient_PublishSubscribe_1000` publishes 1000 messages and verifies all are received via `Subscription.Poll` using an embedded driver
- [ ] **F010-T04**: Test `TestClient_ErrDriverUnavailable` verifies: construct Client against a CNC file with a stale heartbeat; verify `ErrDriverUnavailable` is returned
- [ ] **F010-T05**: Test `TestPublication_ErrBackPressure` fills the log buffer and verifies `Offer` returns `ErrBackPressure`
- [ ] **F010-T06**: `examples/basic_publisher/main.go` and `examples/basic_subscriber/main.go` compile: `go build ./examples/...` exits 0
