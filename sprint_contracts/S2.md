## 1. Sprint Description

Sprint S2 delivers the QUIC transport layer that sits between the driver agents and the network. By the end of this sprint, Hyperspace can open N concurrent QUIC connections to a remote peer (each enforcing TLS 1.3 with mTLS), pool those connections in a thread-safe data structure, and select the best connection per send batch using the LowestRTT arbitration strategy. The QUIC adapter wraps quic-go behind a `Connection` interface so the upper layers never import quic-go types directly.

## 3. Acceptance Criteria (Must Pass)

### 3.1 Feature F-003: QUIC Transport Adapter

#### Interface definition

- [ ] **F003-C01**: Package `pkg/transport/quic` compiles — `go build ./pkg/transport/quic/...` exits 0
- [ ] **F003-C02**: Package exports interface `Connection` with exactly these methods:
  - `Send(frames []Frame) error`
  - `Recv() (Frame, error)`
  - `Close() error`
  - `RTT() time.Duration`
  - `RemoteAddr() net.Addr`
  - `ID() string`
- [ ] **F003-C03**: Package exports `Frame` struct with fields: `StreamID int32`, `SessionID int32`, `Flags uint8`, `Payload []byte`
- [ ] **F003-C04**: Package exports interface `Listener` with methods: `Accept(ctx context.Context) (Connection, error)`, `Close() error`, `Addr() net.Addr`

#### Dial and Listen

- [ ] **F003-F01**: `Dial(ctx context.Context, addr string, tlsCfg *tls.Config) (Connection, error)` opens a QUIC connection to `addr`; returns a `Connection` on success; returns a wrapped error on failure
- [ ] **F003-F02**: `Listen(ctx context.Context, addr string, tlsCfg *tls.Config) (Listener, error)` binds a QUIC listener on `addr`; `Listener.Accept` blocks until a new connection arrives
- [ ] **F003-F03**: All QUIC connections (both Dial and Listen) set `MinVersion: tls.VersionTLS13` — no TLS version below 1.3 is negotiated
- [ ] **F003-F04**: QUIC connection parameters set: `MaxIncomingStreams: 4096`, `KeepAlivePeriod: 5 * time.Second`
- [ ] **F003-F05**: `Connection.Send(frames)` serialises frames and writes them to a QUIC stream; the stream is reused across multiple `Send` calls for the same connection (not a new stream per send)
- [ ] **F003-F06**: `Connection.Recv()` reads the next frame from the QUIC stream; returns `io.EOF` when the connection is closed cleanly

#### TLS configuration helper

- [ ] **F003-F07**: `NewTLSConfig(cert tls.Certificate, rootCAs *x509.CertPool, requireClientCert bool) *tls.Config` returns a `*tls.Config` with `MinVersion: tls.VersionTLS13`, the provided cert, and `ClientAuth: tls.RequireAndVerifyClientCert` when `requireClientCert` is true

#### Integration test

- [ ] **F003-T01**: `go test -race -count=1 ./pkg/transport/quic/...` exits 0
- [ ] **F003-T02**: Integration test `TestQUIC_SendRecv_1000Frames` starts an in-process Listener and Dial pair on `127.0.0.1:0`; sends 1,000 frames; verifies all 1,000 frames received in order with correct StreamID and Payload content
- [ ] **F003-T03**: Integration test `TestQUIC_TLS13Enforced` verifies that a connection attempt with `MaxVersion: tls.VersionTLS12` on the server side fails with a TLS error (not a silent downgrade)
- [ ] **F003-T04**: `go test -coverprofile=cover.out ./pkg/transport/quic/...` produces ≥ 90% statement coverage

### 3.2 Feature F-004: Multi-QUIC Connection Pool

#### Pool structure

- [ ] **F004-C01**: Package `pkg/transport/pool` compiles — exit code 0
- [ ] **F004-C02**: Package exports `Pool` struct (or type — may be an interface with a concrete implementation)
- [ ] **F004-C03**: `Pool` has exported methods: `Add(conn quic.Connection)`, `Remove(id string)`, `Connections() []quic.Connection`, `Len() int`

  > Note: `quic.Connection` here refers to `pkg/transport/quic.Connection` (the Hyperspace interface), not the quic-go type.

#### Thread safety

- [ ] **F004-F01**: `Pool.Add` and `Pool.Remove` acquire a write lock (`sync.RWMutex`) — concurrent calls do not corrupt the pool
- [ ] **F004-F02**: `Pool.Connections()` acquires a read lock and returns a snapshot (a new slice) — callers can iterate the snapshot without holding any lock
- [ ] **F004-F03**: `Pool.Len()` is safe to call without any external lock

#### Pool construction

- [ ] **F004-F04**: `NewPool(target int) *Pool` creates a pool with a target size of `target` connections; `target` is advisory (Pool Manager enforces actual sizing)
- [ ] **F004-F05**: `Pool.Target() int` returns the configured target size

#### Tests

- [ ] **F004-T01**: `go test -race -count=1 ./pkg/transport/pool/...` exits 0
- [ ] **F004-T02**: Test `TestPool_ConcurrentAddRemove` spawns 8 goroutines that concurrently call `Add` and `Remove` on the same pool; race detector passes; final `Len()` is consistent with the number of surviving connections
- [ ] **F004-T03**: `go test -coverprofile=cover.out ./pkg/transport/pool/...` produces ≥ 90% statement coverage

### 3.3 Feature F-005: Connection Arbitrator

#### Strategy enum and interface

- [ ] **F005-C01**: Package `pkg/transport/arbitrator` compiles — exit code 0
- [ ] **F005-C02**: Package exports type `Strategy` (string or int enum) with constants: `LowestRTT`, `LeastOutstanding`, `Hybrid`, `Sticky`, `Random`
- [ ] **F005-C03**: Package exports `Arbitrator` struct (or interface) with method: `Select(strategy Strategy) (quic.Connection, error)`
  > `quic.Connection` is `pkg/transport/quic.Connection`
- [ ] **F005-C04**: `ErrEmptyPool` is exported as a sentinel error value returned by `Select` when the pool has no connections

#### Strategy implementations

- [ ] **F005-F01**: `LowestRTT` — `Select` returns the connection with the minimum `RTT()` value from the pool snapshot
- [ ] **F005-F02**: `LeastOutstanding` — `Select` returns the connection with the lowest outstanding-frame count (tracked as `int64` per connection via `sync/atomic`)
- [ ] **F005-F03**: `Hybrid` — `Select` computes a weighted score `score = 0.7 * normalisedRTT + 0.3 * normalisedOutstanding` and returns the connection with the lowest score; normalisation divides each value by the max across the pool
- [ ] **F005-F04**: `Sticky` — `Select` returns the connection returned by the previous `Select` call if it is still in the pool and `RTT() < 2 * minRTT`; otherwise falls back to `LowestRTT`
- [ ] **F005-F05**: `Random` — `Select` returns a uniformly random connection from the pool using `math/rand/v2`
- [ ] **F005-F06**: All strategy implementations return `ErrEmptyPool` when `pool.Len() == 0`

#### Arbitrator construction

- [ ] **F005-F07**: `NewArbitrator(pool *pool.Pool) *Arbitrator` creates an Arbitrator bound to the given pool
- [ ] **F005-F08**: Arbitrator is safe to call concurrently from multiple goroutines (Sender agent calls Select from a hot loop)

#### Tests

- [ ] **F005-T01**: `go test -race -count=1 ./pkg/transport/arbitrator/...` exits 0
- [ ] **F005-T02**: Test `TestArbitrator_LowestRTT` creates a pool with 4 mock connections with RTTs `[10ms, 5ms, 20ms, 15ms]`; verifies `Select(LowestRTT)` returns the 5ms connection
- [ ] **F005-T03**: Test `TestArbitrator_ErrEmptyPool` calls `Select` on an empty pool; verifies `ErrEmptyPool` is returned
- [ ] **F005-T04**: Test `TestArbitrator_Sticky_FallsBack` verifies Sticky falls back to LowestRTT when the previously selected connection is removed from the pool
- [ ] **F005-T05**: Test `TestArbitrator_Hybrid_Deterministic` verifies Hybrid produces a deterministic result given fixed RTT and outstanding values
- [ ] **F005-T06**: `go test -coverprofile=cover.out ./pkg/transport/arbitrator/...` produces ≥ 90% statement coverage
- [ ] **F005-T07**: Benchmark `BenchmarkArbitrator_LowestRTT` with 8 connections completes in < 500 ns per op (verified with `go test -bench=BenchmarkArbitrator_LowestRTT -benchtime=5s`)

## 5. Holdout Scenarios

| Scenario ID | Description | Expected Outcome |
|---|---|---|
| S2-H01 | `Dial` called with an unreachable address and a 1-second context deadline | Returns a context deadline error within 1.1 seconds; no goroutine leak (goleak) |
| S2-H02 | Server closes connection while client is mid-`Send` | `Send` returns a non-nil error; client code can detect and handle gracefully |
| S2-H03 | `Arbitrator.Select(LowestRTT)` with 4 connections; one connection's `RTT()` updates to `MaxInt64` (simulating probe timeout) | The MaxInt64 connection is never selected; another connection is always returned |
| S2-H04 | `Pool.Add` called with the same connection ID twice | Second `Add` is a no-op or replaces the existing entry — no duplicate entries; `Len()` reflects accurate count |
| S2-H05 | TLS handshake with client presenting a self-signed cert not in `rootCAs` | Handshake fails; `Dial` returns a TLS error; connection is rejected |
| S2-H06 | `Arbitrator.Select(Random)` called 10,000 times on a 4-connection pool | All 4 connections selected at least once (chi-squared test at p=0.01 significance) |

Minimum satisfaction score to pass: **0.95**
