### F-030: DRL Fallback to BBRv3 (Critical — ADR-003 conformance)

**Acceptance Criteria:**

- [ ] ADR-007 recorded in `decision_log.md`

### F-031: SVID Continuous Rotation (Critical — ADR-004 conformance)

**Acceptance Criteria:**
- [ ] `SPIFFESource` starts a background goroutine with `WatchX509Context`
- [ ] `OnX509ContextUpdate` atomically updates stored `*tls.Config` via `sync/atomic.Pointer`
- [ ] `TLSConfig()` returns the latest live config
- [ ] Background watcher stops cleanly on context cancellation
- [ ] Test: initial fetch works
- [ ] Test: config update is applied atomically

- [ ] ADR-008 recorded in `decision_log.md`

### F-032: Send Retry with Back-Pressure Propagation (Critical — data integrity)

**File:** `pkg/driver/sender/sender.go`

**Acceptance Criteria:**
- [ ] On `conn.Send()` failure, retry on next connection (up to 3 attempts, round-robin)
- [ ] After 3 failed attempts: increment `CtrLostFrames`, log at `slog.Error`
- [ ] Per-publication failure counter tracks consecutive failures
- [ ] After 3 consecutive failed frames, back-pressure flag is set
- [ ] Failure counter resets on successful send
- [ ] Test: single-connection failure retries on another
- [ ] Test: all-connections-fail increments `CtrLostFrames`
- [ ] Test: back-pressure propagation after consecutive failures
- [ ] ADR-009 recorded in `decision_log.md`

### F-033: Inbound TLS and SPIFFE ID Validation (Critical — security)

**File:** `pkg/transport/quic/conn.go`

**Acceptance Criteria:**
- [ ] `Accept()` verifies `HandshakeComplete == true`
- [ ] `Accept()` verifies `PeerCertificates` is non-empty
- [ ] `RequireSPIFFE bool` field added to accept config
- [ ] When `RequireSPIFFE` is true, peer cert must contain `spiffe://` URI SAN
- [ ] Test: valid SPIFFE cert accepted
- [ ] Test: missing cert rejected
- [ ] Test: non-SPIFFE cert rejected when RequireSPIFFE=true
- [ ] ADR-010 recorded in `decision_log.md`
