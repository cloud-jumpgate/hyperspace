## 1. Sprint Description

Sprint S9 delivers full AWS integration and SPIFFE/SPIRE workload identity. By end of sprint, hsd discovers peers from Cloud Map, loads configuration from SSM Parameter Store, fetches TLS certificates from Secrets Manager, and obtains short-lived X.509 SVIDs from the SPIRE Agent with automatic rotation watching. All AWS SDK calls use IAM role credentials. Localstack integration tests run in CI. Test coverage is ≥ 80% for AWS packages, ≥ 85% for identity packages.

## 3. Acceptance Criteria (Must Pass)

### 3.1 Feature F-013: AWS Cloud Map Resolver

- [ ] **F013-CM01**: Package `pkg/discovery` compiles — exit code 0
- [ ] **F013-CM02**: `CloudMapResolver` struct with `Resolve(ctx context.Context, serviceName string) ([]string, error)` returning `host:port` strings
- [ ] **F013-CM03**: `Resolve` uses `aws-sdk-go-v2/service/servicediscovery` — no hardcoded endpoints
- [ ] **F013-CM04**: `Resolve` returns wrapped error on API failure — never panics
- [ ] **F013-CM05**: Localstack integration test: start Cloud Map namespace, register 3 instances, verify `Resolve` returns all 3 endpoints
- [ ] **F013-CM06**: `pkg/discovery` test coverage ≥ 80%

### 3.2 Feature F-013: SSM Configuration

- [ ] **F013-SSM01**: Package `pkg/config` compiles — exit code 0
- [ ] **F013-SSM02**: `SSMConfig.Get(ctx context.Context, paramPath string) (string, error)` fetches a single SSM parameter by path
- [ ] **F013-SSM03**: `SSMConfig.GetOrDefault(ctx context.Context, paramPath string, defaultValue string) string` returns `defaultValue` when SSM returns `ParameterNotFound` — allows partial SSM deployments
- [ ] **F013-SSM04**: SSM client uses `aws-sdk-go-v2/service/ssm` with IAM role credentials — no hardcoded keys
- [ ] **F013-SSM05**: Localstack integration test: write a parameter to SSM, verify `Get` returns the value; verify `GetOrDefault` returns default when parameter absent
- [ ] **F013-SSM06**: `pkg/config` test coverage ≥ 80%

### 3.3 Feature F-013: Secrets Manager Certificate Loader

- [ ] **F013-SM01**: `SecretsManagerCertLoader` in `pkg/identity` with `Load(ctx context.Context, arn string) (*tls.Certificate, error)`
- [ ] **F013-SM02**: `Load` fetches the PEM-encoded cert+key bundle from Secrets Manager by ARN using `aws-sdk-go-v2/service/secretsmanager`
- [ ] **F013-SM03**: `Load` parses the PEM bundle and returns a `*tls.Certificate` for direct use in `tls.Config`
- [ ] **F013-SM04**: No credentials hardcoded in source — verified by `gosec` passing
- [ ] **F013-SM05**: Localstack integration test: store a self-signed PEM bundle in Secrets Manager; verify `Load` returns a valid `*tls.Certificate`

### 3.4 Feature F-014: SPIFFE/SPIRE Identity

- [ ] **F014-S01**: `SPIFFEIdentity` struct in `pkg/identity` with `FetchSVID(ctx context.Context) (*tls.Certificate, *x509.CertPool, error)`
- [ ] **F014-S02**: `FetchSVID` connects to SPIRE Agent workload API at `unix:///run/spire/agent.sock` (path configurable via SSM param `hyperspace/spire/socket_path`)
- [ ] **F014-S03**: `SPIFFEIdentity.Watch(ctx context.Context, onChange func(*tls.Certificate, *x509.CertPool))` subscribes to SVID rotation events and calls `onChange` for each new SVID before expiry
  NOTE (ADR-008, 2026-04-19): The Watch callback API specified in F014-S03 was
  superseded by StartWatch() + atomic.Pointer[tls.Config] as documented in ADR-008.
  The StartWatch pattern was accepted by the Architecture Evaluator (see F-031 in
  progress.json). This criterion is satisfied by the ADR-008 implementation.
- [ ] **F014-S04**: SVID serialisation uses `svid.Marshal()` which produces PKCS#8 format — not `x509.MarshalECPrivateKey` (which produces SEC1/PKCS#1 format incompatible with `tls.X509KeyPair`)
- [ ] **F014-S05**: Integration test: mock SPIRE Agent using go-spiffe test utilities; `FetchSVID` returns a valid `*tls.Certificate` with SPIFFE URI SAN
- [ ] **F014-S06**: Integration test: mock SPIRE Agent rotates the SVID; `Watch` callback fires with new cert within 100 ms
- [ ] **F014-S07**: `pkg/identity` test coverage ≥ 85%
