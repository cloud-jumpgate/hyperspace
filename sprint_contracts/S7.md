## 1. Sprint Description

Sprint S7 delivers the full congestion control suite: CUBIC (RFC 8312), BBR v1, BBRv3, and a DRL controller backed by ONNX Runtime via CGO. All controllers implement the `CongestionController` interface. The DRL controller falls back to BBRv3 when the ONNX model is unavailable. The `//go:build onnx` tag gates CGO compilation. Test coverage is ≥ 85%.

## 3. Acceptance Criteria (Must Pass)

### 3.1 CongestionController Interface

- [ ] **F011-I01**: Package `pkg/cc` compiles — exit code 0
- [ ] **F011-I02**: `CongestionController` interface defined with methods: `OnAck(ackedBytes int64, rtt time.Duration)`, `OnLoss(lostBytes int64)`, `CongestionWindow() int64`
- [ ] **F011-I03**: All four controller types implement `CongestionController` — verified by compile-time interface assertion

### 3.2 CUBIC

- [ ] **F011-C01**: Package `pkg/cc/cubic` compiles — exit code 0
- [ ] **F011-C02**: `NewCUBIC(initialCwnd int64) *CUBIC` constructs a CUBIC controller
- [ ] **F011-C03**: CUBIC `CongestionWindow()` returns values in bytes (not packets)
- [ ] **F011-C04**: CUBIC implements the cubic growth function `W_cubic(t) = C*(t - K)^3 + W_max` per RFC 8312 §5
- [ ] **F011-C05**: CUBIC multiplies cwnd by `beta_cubic = 0.7` on loss event per RFC 8312 §5.8
- [ ] **F011-C06**: `pkg/cc/cubic` test coverage ≥ 85%

### 3.3 BBRv1

- [ ] **F011-B1-01**: Package `pkg/cc/bbr` compiles — exit code 0
- [ ] **F011-B1-02**: `NewBBR(initialCwnd int64) *BBR` constructs a BBR v1 controller
- [ ] **F011-B1-03**: BBR tracks `btlBw` (bottleneck bandwidth estimate) and `rtProp` (minimum RTT) per the BBR paper
- [ ] **F011-B1-04**: BBR sets `cwnd = btlBw * rtProp * gain` where gain is state-machine dependent
- [ ] **F011-B1-05**: `pkg/cc/bbr` test coverage ≥ 85%

### 3.4 BBRv3

- [ ] **F011-B3-01**: Package `pkg/cc/bbrv3` compiles — exit code 0
- [ ] **F011-B3-02**: `NewBBRv3(initialCwnd int64) *BBRv3` constructs a BBRv3 controller
- [ ] **F011-B3-03**: BBRv3 refines loss handling vs BBRv1: does not reduce cwnd on every loss (only when loss rate exceeds threshold) per the Google BBRv3 proposal
- [ ] **F011-B3-04**: `pkg/cc/bbrv3` test coverage ≥ 85%

### 3.5 DRL Controller

- [ ] **F011-D01**: Package `pkg/cc/drl` compiles without `//go:build onnx` — fallback stub only, exit code 0
- [ ] **F011-D02**: With `//go:build onnx`, `pkg/cc/drl` compiles against ONNX Runtime CGO binding
- [ ] **F011-D03**: `NewDRLController(modelPath string, fallback CongestionController) (*DRLController, error)` loads the ONNX model from `modelPath`
- [ ] **F011-D04**: DRL input state vector: `[rtt_ms float32, loss_rate float32, cwnd_bytes float32, outstanding_bytes float32]` (4 elements)
- [ ] **F011-D05**: DRL output: `cwnd_delta float32` — applied to current cwnd as `cwnd += int64(cwnd_delta)`
- [ ] **F011-D06**: DRL automatically falls back to `fallback` controller if: model fails to load, inference returns error, or more than 5 inference errors occur in a 60-second window
- [ ] **F011-D07**: `pkg/cc/drl` test coverage ≥ 85% excluding CGO paths (CGO paths tested with build tag integration test)

### 3.6 Tests

- [ ] **F011-T01**: `go test -race -count=1 ./pkg/cc/...` exits 0 (without onnx tag)
- [ ] **F011-T02**: Combined `pkg/cc/` coverage ≥ 85%
- [ ] **F011-T03**: Test `TestCUBIC_LossResponse` verifies: after a loss event, cwnd is reduced to 70% of the pre-loss value
- [ ] **F011-T04**: Test `TestDRL_FallbackOnLoadError` verifies: construct DRLController with a non-existent model path; verify all OnAck/OnLoss/CongestionWindow calls delegate to fallback controller

## 7. Holdout Scenarios

| Scenario ID | Description | Expected Outcome |
|---|---|---|
| S7-H01 | CUBIC controller receives 100 consecutive acks then a loss | cwnd grows per cubic function, then drops to 70% on loss |
| S7-H02 | DRL inference returns NaN | DRL logs warning; increments error counter; falls back to BBRv3 for that call |
| S7-H03 | DRL inference fails 5 times in 60 s | Controller permanently switches to fallback for the rest of the window |
| S7-H04 | `cc=cubic` channel URI param | Arbitrator uses CUBIC controller for that connection |
