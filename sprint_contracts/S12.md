## 1. Sprint Description

Sprint S12 addresses fault tolerance gaps and wires congestion control algorithms into QUIC connections. The pool manager has no reconnection logic (A-02) -- pool drains to zero on network blips. CC algorithms exist but are never applied to QUIC connections (F-03). Sender position tracking breaks on term rotation (S-03). SessionID collision probability is too high with 32-bit random IDs (C-02).

## 4. Acceptance Criteria (Must Pass)

### 4.1 Feature F-020: Pool Reconnection Logic (A-02)

- [ ] **F-020-C01**: `pkg/driver/poolmgr/poolmgr.go` compiles without errors
- [ ] **F-020-F01**: PoolManager runs a health-check loop (default 500ms interval) that detects closed connections
- [ ] **F-020-F02**: When a closed connection is detected, it is removed from the pool and a reconnection attempt is made with exponential backoff (starting at 100ms, max 10s)
- [ ] **F-020-F03**: When pool size reaches 0, `ErrNoConnections` is surfaced (not silently ignored)
- [ ] **F-020-F04**: Reconnection attempts are bounded (configurable max retries, default 5 per health check cycle)
- [ ] **F-020-T01**: `TestHealthCheck_DetectsClosedConnection` -- close a mock connection, verify health check removes it and attempts reconnect
- [ ] **F-020-T02**: `TestHealthCheck_ExponentialBackoff` -- verify backoff timing increases on consecutive failures
- [ ] **F-020-T03**: `TestHealthCheck_PoolDrainedToZero_SurfacesError` -- verify ErrNoConnections is logged when pool is empty

### 4.2 Feature F-021: CC Algorithm Adapter (F-03)

- [ ] **F-021-C01**: New file `pkg/cc/adapter.go` exists and compiles
- [ ] **F-021-F01**: `CCAdapter` struct implements `quic-go`'s `CongestionControlAlgorithm` interface (or equivalent integration point) and delegates to the Hyperspace `cc.CongestionControl` interface
- [ ] **F-021-F02**: The adapter is wired at connection creation time -- when `ccName` is passed to `quictr.Dial`, the appropriate CC algorithm is instantiated and applied
- [ ] **F-021-F03**: If ccName is empty or unknown, CUBIC is used as the default (not no-CC)
- [ ] **F-021-T01**: `TestCCAdapter_DelegatesToHyperspaceCC` -- verify OnPacketSent/Acked/Lost callbacks are forwarded
- [ ] **F-021-T02**: `TestDial_AppliesCCAlgorithm` -- verify that dialing with ccName="bbr" results in BBR being active

### 4.3 Feature F-022: Sender Position Term-Aware Tracking (S-03)

- [ ] **F-022-C01**: `pkg/driver/sender/sender.go` compiles without errors
- [ ] **F-022-F01**: Position tracking uses `(partitionIndex, termOffset)` tuple instead of a single int64 offset
- [ ] **F-022-F02**: When the active partition changes (term rotation), the termOffset resets to 0 for the new partition
- [ ] **F-022-F03**: Frames are not re-sent or skipped after term rotation
- [ ] **F-022-T01**: `TestDoWork_PositionTracksPartitionChange` -- rotate partition, verify sender reads from offset 0 in new partition
- [ ] **F-022-T02**: `TestDoWork_NoFrameSkipOnRotation` -- fill partition, rotate, write new frames, verify all are sent

### 4.4 Feature F-023: Composite Session Key (C-02)

- [ ] **F-023-C01**: `pkg/driver/receiver/receiver.go` compiles without errors
- [ ] **F-023-F01**: Image map uses composite key `(sessionID int32, streamID int32)` as uint64 instead of sessionID alone
- [ ] **F-023-F02**: Two publishers with different streamIDs but same sessionID get separate image buffers
- [ ] **F-023-T01**: `TestCompositeKey_SameSessionDifferentStream` -- verify separate images created
- [ ] **F-023-T02**: `TestCompositeKey_SameSessionSameStream_SharedImage` -- verify same image reused

- [ ] **NF-02**: Test coverage >= 90% for modified packages
