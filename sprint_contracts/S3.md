### 3.2 Feature F-008: Sender Agent

#### Compilation and structure

- [ ] **F008-S01**: Package `pkg/driver/sender` compiles â€” exit code 0
- [ ] **F008-S02**: `Sender` struct exported with `Run(ctx context.Context) error` method

#### Send loop

- [ ] **F008-S03**: Sender busy-polls all active publication log buffers using `Reader.Poll` with a configurable `maxBatchSize` (default 64 frames)
- [ ] **F008-S04**: For each batch of frames polled, Sender calls `Arbitrator.Select` once per batch (not once per frame) to choose the outbound QUIC connection
- [ ] **F008-S05**: If `Arbitrator.Select` returns `ErrEmptyPool`, Sender backs off with a short sleep (configurable, default 1 ms) and retries â€” does not drop frames
- [ ] **F008-S06**: Sender yields the CPU (`runtime.Gosched()`) between poll iterations when no frames are available, to avoid burning a core at 100% when idle

#### Lifecycle

- [ ] **F008-S07**: `Sender.Run` returns when `ctx` is cancelled â€” no goroutine leak
- [ ] **F008-S08**: `goleak.VerifyNone(t)` passes in all Sender tests
