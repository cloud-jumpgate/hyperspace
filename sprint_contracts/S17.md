## F-039: Implement DEF-005 — SVID Cert Rotation in PoolManager

**Feature:** F-009 Pool Manager Agent
**Condition:** Genuine functional gap — PoolManager does not subscribe to SVID rotation events or perform drain-then-close on new cert.
**Package:** `pkg/driver/poolmgr`
**Security-required:** Yes — Security Evaluator PASS required before `evaluator_pass`

### Background

The `pkg/identity` package provides `SPIFFESource` which calls `StartWatch(ctx, callback)`. The callback is invoked with a new `*tls.Config` whenever SPIRE issues a fresh SVID. Current PoolManager stores a static `tlsConf *tls.Config` and never reacts to SVID rotation.

Production impact: after SVID expiry (typically 1 hour), all existing QUIC connections will fail TLS handshake on reconnect. New connections opened after expiry will use expired credentials.

### Required Implementation

#### Interface (add to `pkg/driver/poolmgr/poolmgr.go`)

```go
// SVIDWatcher is implemented by pkg/identity.SPIFFESource.
// OnRotation is called with the new TLS config whenever a fresh SVID is received.
type SVIDWatcher interface {
    StartWatch(ctx context.Context, callback func(newTLS *tls.Config)) error
}
```

#### PoolManager struct changes

- Add `svid SVIDWatcher` field (optional — nil = no rotation)
- Add `mu sync.RWMutex` protecting `tlsConf` (read on every new dial, write on rotation)
- Add `pendingClose []quictr.Connection` — connections to drain and close after rotation

#### Rotation flow (`rotateCerts` method)

```
1. Build new TLS config (received via callback)
2. Lock mu for write; update mgr.tlsConf; unlock
3. Snapshot current pool connections into pendingClose
4. Open MinSize new connections using new TLS config (EnsureMinConnections)
5. For each connection in pendingClose: call Close() to drain and close it
6. Remove closed connections from pool
```

#### Run loop integration

- If `mgr.svid != nil`, call `mgr.svid.StartWatch(ctx, mgr.rotateCerts)` in `Run`
- `rotateCerts` must be safe to call from the watcher goroutine (use mu)

#### New constructor

```go
func NewWithSVID(
    p *pool.Pool,
    pm *pathmgr.PathManager,
    learner *AdaptivePoolLearner,
    svid SVIDWatcher,
    peerAddr, ccName string,
    learnerInterval time.Duration,
    dialer Dialer,
) *PoolManager
```

#### Tests

- `TestPoolManager_CertRotation_DrainsThenCloses`: inject a mock `SVIDWatcher` that fires a rotation callback; assert old connections are closed and new connections are opened using the new TLS config
- `TestPoolManager_CertRotation_NoDeadlock`: fire rotation under concurrent DoWork ticks; pass `-race`

### Acceptance Criteria (F-039)

- [ ] `SVIDWatcher` interface defined in `pkg/driver/poolmgr`
- [ ] `PoolManager.rotateCerts` method implemented
- [ ] `NewWithSVID` constructor added
- [ ] Old connections drained and closed after rotation
- [ ] New connections opened with new TLS config before old are closed
- [ ] `mu sync.RWMutex` protects `tlsConf` access; no data race
- [ ] `TestPoolManager_CertRotation_DrainsThenCloses` passes
- [ ] `TestPoolManager_CertRotation_NoDeadlock` passes with `-race`
- [ ] `go test -race ./pkg/driver/poolmgr/` exits 0

1. System architecture conformance to `SYSTEM_ARCHITECTURE.md`
