## 3. Acceptance Criteria (Must Pass)

### 3.1 GitHub Actions CI

- [ ] **F-S10-CI01**: `.github/workflows/ci.yml` exists and is valid YAML
- [ ] **F-S10-CI02**: CI pipeline triggers on `push` to any branch and on `pull_request` to `main`
- [ ] **F-S10-CI03**: CI runs `go test -race ./...` — pipeline fails if any test fails
- [ ] **F-S10-CI04**: CI runs `golangci-lint run` using the `.golangci.yml` config — pipeline fails on lint errors
- [ ] **F-S10-CI05**: CI runs `gosec ./...` — pipeline fails on any high-severity finding
- [ ] **F-S10-CI06**: CI runs `govulncheck ./...` — pipeline fails on known vulnerabilities
- [ ] **F-S10-CI07**: CI produces a test coverage report; fails if total coverage < 85%
- [ ] **F-S10-CI08**: CI builds the `hsd` binary with `go build ./cmd/hsd` — pipeline fails if build fails
- [ ] **F-S10-CI09**: Localstack Docker Compose service runs in CI for AWS integration tests (`pkg/discovery`, `pkg/config`, `pkg/identity`)
- [ ] **F-S10-CI10**: CI caches Go module downloads and build cache across runs for speed

### 3.2 golangci.yml Configuration

- [ ] **F-S10-L01**: `.golangci.yml` exists and enables minimum linters: `errcheck`, `staticcheck`, `gosimple`, `unused`, `govet`, `ineffassign`, `revive`
- [ ] **F-S10-L02**: `.golangci.yml` excludes `pkg/cc/drl` from CGO-related false positives when run without onnx tag
- [ ] **F-S10-L03**: `golangci-lint run` with this config exits 0 on the clean codebase

### 3.3 Makefile

- [ ] **F-S10-MK01**: `Makefile` exists with targets: `test`, `test-cover`, `bench`, `lint`, `vuln`, `sec`, `build`, `build-hsd`, `docker-build`, `docker-run`, `localstack-up`, `localstack-down`, `help`
- [ ] **F-S10-MK02**: `make test` runs `go test -race ./...`
- [ ] **F-S10-MK03**: `make test-cover` runs `go test -race -coverprofile=cover.out ./...` and prints total coverage
- [ ] **F-S10-MK04**: `make build` builds `hsd`, `hyperspace-stat`, and `hyperspace-probe` binaries to `./bin/`
- [ ] **F-S10-MK05**: `make help` prints all targets with descriptions
- [ ] **F-S10-MK06**: `make lint` invokes `golangci-lint run`
- [ ] **F-S10-MK07**: `make localstack-up` starts the Localstack container via `docker compose up -d`

### 3.4 Dockerfile

- [ ] **F-S10-DF01**: `Dockerfile` exists at repo root
- [ ] **F-S10-DF02**: Multi-stage build: builder stage uses `golang:1.26-alpine`, runtime stage uses `gcr.io/distroless/static:nonroot`
- [ ] **F-S10-DF03**: Builder compiles `hsd` with `CGO_ENABLED=0 GOOS=linux GOARCH=arm64` for the c7gn.4xlarge target
- [ ] **F-S10-DF04**: Runtime image contains only the `hsd` binary — no source code, no Go toolchain
- [ ] **F-S10-DF05**: Container runs as non-root user (distroless nonroot default UID 65532)
- [ ] **F-S10-DF06**: `docker build -t hyperspace-hsd .` completes without error

### 3.5 Benchmarks

- [ ] **F-S10-BN01**: `cmd/hyperspace-probe/main.go` exists and compiles — benchmark tool entry point
- [ ] **F-S10-BN02**: `pkg/logbuffer` has `BenchmarkAppender` benchmark that can be run with `go test -bench=BenchmarkAppender ./pkg/logbuffer/`
- [ ] **F-S10-BN03**: `pkg/transport/arbitrator` has `BenchmarkArbitrator_LowestRTT` with an 8-connection pool
- [ ] **F-S10-BN04**: `make bench` runs all benchmarks and produces output — does not fail on performance numbers (benchmarks are informational)

### 3.6 README

- [ ] **F-S10-RM01**: `README.md` exists at repo root
- [ ] **F-S10-RM02**: README sections: Project Purpose, Architecture Overview (one paragraph), Prerequisites, Quick Start, Configuration (channel URI format), Contributing, Licence
- [ ] **F-S10-RM03**: Quick Start section: a new engineer can run `make test` and see tests pass within the steps described
- [ ] **F-S10-RM04**: Prerequisites listed: Go 1.26+, Docker (for Localstack), golangci-lint, SPIRE Agent (production only)
- [ ] **F-S10-RM05**: Channel URI format documented with example: `hs:quic?endpoint=10.0.0.1:7070|pool=4|cc=bbrv3`
