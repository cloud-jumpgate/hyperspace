## 1. Sprint Description

Sprint S5 delivers two cooperating components: the `AdaptivePoolLearner`, which analyses ProbeRecord metrics and recommends pool size adjustments, and the `PoolManager` agent, which executes those recommendations by opening and closing QUIC connections. By end of sprint, the pool self-sizes between `minPoolSize` and `maxPoolSize` based on observed RTT-loss correlation. Test coverage is ≥ 90% for AdaptivePoolLearner and ≥ 85% for PoolManager.

## 3. Acceptance Criteria (Must Pass)

### 3.1 Feature F-007: Adaptive Pool Learner

- [ ] **F007-C01**: `AdaptivePoolLearner` struct exported in `pkg/driver/poolmgr` — compiles exit code 0
- [ ] **F007-C02**: `AdaptivePoolLearner.Evaluate(metrics []ProbeRecord) PoolSizeRecommendation` is a pure function — no side effects, no I/O
- [ ] **F007-F01**: `PoolSizeRecommendation` struct has fields: `TargetSize int`, `Reason string`, `Action PoolAction` (enum: Add, Remove, Hold)
- [ ] **F007-F02**: Learner recommends `Add` when: mean RTT improves by > 10% comparing current pool to current pool + 1 connection (using probe history) AND current size < `maxPoolSize`
- [ ] **F007-F03**: Learner recommends `Remove` when: loss rate < 0.1% (1000 PPM) AND all connections have shown no RTT improvement benefit for > 60 seconds AND current size > `minPoolSize`
- [ ] **F007-F04**: Learner recommends `Hold` in all other cases — default safe behaviour
- [ ] **F007-F05**: Pool size bounded to `[minPoolSize, maxPoolSize]` — no recommendation outside this range
- [ ] **F007-F06**: Learner is constructable with configurable `minPoolSize`, `maxPoolSize`, and `evaluationWindowSec` parameters

#### Tests for AdaptivePoolLearner

- [ ] **F007-T01**: `go test -race -count=1 ./pkg/driver/poolmgr/...` exits 0
- [ ] **F007-T02**: AdaptivePoolLearner test coverage ≥ 90%
- [ ] **F007-T03**: Test `TestLearner_RecommendsAdd` verifies: inject probe records where adding a connection reduces mean RTT by 15%; expect `Action == Add`
- [ ] **F007-T04**: Test `TestLearner_RecommendsRemove` verifies: inject probe records with loss < 1000 PPM, stable RTT for 65 s; expect `Action == Remove`
- [ ] **F007-T05**: Test `TestLearner_HoldAtMax` verifies: inject metrics that would trigger Add but size is already at `maxPoolSize`; expect `Action == Hold`
- [ ] **F007-T06**: Test `TestLearner_HoldAtMin` verifies: inject metrics that would trigger Remove but size is already at `minPoolSize`; expect `Action == Hold`

### 3.2 Feature F-009: Pool Manager Agent

- [ ] **F009-C01**: `PoolManager` struct exported in `pkg/driver/poolmgr` with `Run(ctx context.Context) error` method
- [ ] **F009-F01**: On startup, PoolManager opens `targetPoolSize` QUIC connections to each configured peer using a configurable `Dialer` function (injected at construction for testability)
- [ ] **F009-F02**: `Dialer` type defined as `func(ctx context.Context, addr string, tlsCfg *tls.Config) (Connection, error)` — enables test injection without network
- [ ] **F009-F03**: PoolManager calls `AdaptivePoolLearner.Evaluate` every `evaluationIntervalSec` (default 30 s) and executes the recommendation: Add → open a new connection; Remove → close the least-recently-used connection
- [ ] **F009-F04**: Failed connections are detected via a health check; reconnect attempted after `reconnectDelayMs` (default 500 ms) with exponential backoff capped at 30 s
- [ ] **F009-F05**: PoolManager integrates with `pkg/identity` SVID watch: when a new SVID arrives, PoolManager opens new connections with the new cert, drains old connections, then closes them — zero message loss target
- [ ] **F009-F06**: `EnsureMinConnections(n int)` method opens connections synchronously to bring pool to size n — used in startup and test scenarios
- [ ] **F009-F07**: PoolManager shuts down cleanly when ctx is cancelled — closes all pool connections before returning; no goroutine leak

#### Tests for PoolManager

- [ ] **F009-T01**: PoolManager test coverage ≥ 85%
- [ ] **F009-T02**: Test `TestPoolManager_EnsureMin` verifies: inject a mock Dialer; call EnsureMinConnections(4); verify Dialer was called 4 times and pool size is 4
- [ ] **F009-T03**: Test `TestPoolManager_ReconnectOnFailure` verifies: inject a mock Dialer that fails twice then succeeds; verify PoolManager retried and pool is healthy
- [ ] **F009-T04**: Test `TestPoolManager_GracefulShutdown` starts PoolManager with 4 mock connections, cancels ctx, verifies all connections closed and no goroutine leak

## 6. Holdout Scenarios

| Scenario ID | Description | Expected Outcome |
|---|---|---|
| S5-H01 | All 4 connections fail simultaneously | PoolManager retries all 4 with backoff; no panic; reconnects when Dialer recovers |
| S5-H02 | Learner recommends Add; Dialer returns error | PoolManager logs error; retries next evaluation cycle; pool unchanged |
| S5-H03 | SVID rotates while 10,000 messages in-flight | PoolManager opens new-cert connections before closing old; no message loss |
| S5-H04 | minPoolSize == maxPoolSize == 4 | Learner always returns Hold; PoolManager never opens or closes connections |

Minimum satisfaction score to pass: **0.95**

## 8. Actual Outcome Summary

Sprint S5 was completed on 2026-04-17. AdaptivePoolLearner was implemented as a pure function following the RTT-spread and saturation heuristics (design doc §8.4 — see ADR-013 for the policy decision). The `Dialer` injection pattern made PoolManager fully testable without network. Coverage: AdaptivePoolLearner 96.5%, PoolManager 86.2%.

NOTE (2026-04-19, Code Evaluator finding): The S5 sprint outcome summary previously
incorrectly stated that SVID cert rotation (drain-then-close) was implemented and
verified. This was not accurate. The PoolManager (F-009) does not implement SVID
rotation or the drain-then-close flow. This feature was deferred to Sprint S9
(F-014 / pkg/identity) where the SPIFFE/SPIRE identity integration was built.
The pkg/identity StartWatch atomic rotation pattern (ADR-008) handles TLS config
rotation, but integration with PoolManager to drain-then-close connections on cert
rotation remains unimplemented as of S15. This is tracked as DEF-005.
